1. Purpose

Re-edit every onboarding choice without reinstalling, get a notification when a 14-day journey completes, and a bottle-green dark theme for night use.

In plain English: once you finish onboarding today, your language, blocked apps, and intention are frozen — no door back. SP-7 adds that door (Settings), a "journey complete 🎉" notification so completions don't go unnoticed, and a dark mode so the app isn't a white flashlight at night.

3. Scope

In: SettingsScreen + 3 editor screens (reusing onboarding's SelectableCard); themePref + notifyOnCompletion prefs; dark scheme + live theme switch in NiyamTheme; CompletionNotifier; Home gear; routes.

Out: dark overlay (forlater) · daily reminders (YAGNI) · accounts · paywall hooks (SP-6) · re-running the permission flow from settings (Home's protection row already covers it).

4. Architecture

  • UserPrefs.Snapshot + themePref (LIGHT/DARK/SYSTEM, default LIGHT) and notifyOnCompletion (default true); writers setThemePref / setNotifyOnCompletion. Engine reads neither.
  • ui/theme: dark tokens in Color.kt; NiyamTheme resolves the pref from a snapshot-backed Compose state, so switching re-themes live.
  • notifications/CompletionNotifier.kt: channel niyam_completion, PendingIntent → MainActivity; called from maybeComplete, guarded + try/caught (a notification failure never breaks a read).
  • settings/: SettingsScreen (rows + appearance segmented row + toggle + version footer) and Language/BlockedApps/Intention editor screens — pre-selected from snapshot, save → write → pop. Apps editor disables save at zero selections.
  • Routes settings, settings_language, settings_apps, settings_intention; Home gains onOpenSettings.

5. Error handling

Notification failure → caught, read recorded anyway. Permission denied → toggle ON but silent, with a "blocked in system settings" caption. Empty app set → save disabled. Unknown themePref → LIGHT.
