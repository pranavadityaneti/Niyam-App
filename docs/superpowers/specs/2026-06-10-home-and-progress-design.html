1. Purpose

Give Niyam its retention mechanic: every completed unlock-read counts toward the current mantra's 14-day journey, visible on a real home screen (mantra card, Day N of 14, streak, today's reads). Completing a journey triggers a celebration and a curated next-sadhana picker.

In plain English: until now the app blocks and shows; after SP-4 it remembers — every read builds a visible streak and a journey, and finishing a journey hands you the next one.

2. Product rules (locked)

ReadUser tapped Continue after the 15s timer. Any other dismissal ≠ read.

Progress"Day N of M" — N = distinct counted days since the sadhana started; M = completionThresholdDays (14 in v1). Days are the unit (matches schema + the brief's "Day 8 of 14").

3. Scope

In: Room (read_events) + DAO; ProgressRepository + pure ProgressMath (JVM-tested); read recording from the overlay Continue tap (fire-and-forget); UserPrefs += selectedIntention, sadhanaStartEpochDay, completedMantraIds, pendingCelebration; onboarding persists intention + stamps start day; Home rebuilt to brand (mantra card, orange progress bar, streak/today chips, compact protection row); Celebration + next-sadhana picker.

Out: notifications (SP-7) · library + switching + Browse button (SP-5) · threshold tuning UI · timer scaling · stats history · cloud sync.

4. Architecture

  • Room via KSP: one entity ReadEventEntity(id, mantraId, epochDay, timestampMs), indexed.
  • ProgressRepository (object): recordRead() inserts on a background executor then runs the completion check; suspend queries; HomeStats loader.
  • ProgressMath (pure): streak + dayN over epochDay sets — fully unit-tested.
  • The one engine touch: overlay Continue handler records the read (try/catch; unlock can never block or crash on it).
  • Completion: pendingCelebration persisted flag → celebration route → picker (SelectableCard reuse, StarterMantras seeding by stored intention) → new mantra + new start day. "Keep this mantra" restarts the journey.
  • Home: overline "YOUR SADHANA", serif name, script preview, orange progress bar + "Day N of M", streak/today chips, compact protection row (replaces the 4-row panel; tap → permission flow).

5. Error handling

Insert failure → logged, unlock unaffected. Stats failure → zeros, never crash. Set-semantics tolerate clock weirdness. Process death pre-celebration → persisted flag shows it next launch. Unknown ids dropped by curation.
