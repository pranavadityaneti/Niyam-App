1. Purpose

Turn the engineering prototype into the product's first real experience: apply the founder-supplied brand direction app-wide, and build the four-step onboarding that captures intention → starter mantra → language → blocked apps, persists those choices, and feeds them to the blocking engine.

In plain English: today the app works but looks like a lab bench, and it ignores who the user is. After this sub-project it looks like the calm reading app Pranav showed, and the overlay shows your mantra in your language over your chosen apps.

2. Scope and non-goals

In scope

  • NiyamTheme rebuilt from the brand palette + reference aesthetic (light only); applied to all Compose screens.
  • Bundled fonts: Playfair Display (display serif) + Inter (UI sans).
  • Overlay restyled — colors/typography/spacing only; identical ids, structure, timer and unlock logic.
  • Onboarding: Welcome → Intention → Starter mantra → Language → Apps to block → existing 5 permission screens (inherit theme) → Home.
  • Jetpack DataStore persistence: onboardingComplete, currentMantraId, displayLanguage, blockedPackages; warm-up snapshot at app start.
  • CurrentSadhana and BlockList become snapshot-backed (defaults = today's hardcoded values).
  • Returning users route straight to Home.

Explicit non-goals

  • Home redesign / mantra card / progress / streak (SP-4) — Home keeps its permission-status content, restyled by the theme only.
  • Library (SP-5) · Paywall/trial/ads (SP-6) · Dark mode, settings, notifications, re-onboarding (SP-7).
  • Illustrations / iconography sets / deity imagery (none; the no-AI-deity-imagery rule stands) · audio · custom animations.

7. Persistence architecture

  • New dependency: androidx.datastore:datastore-preferences (latest stable 1.1.x, pinned in the version catalog at implementation).
  • UserPrefs: suspend writers + a @Volatile immutable Snapshot(onboardingComplete, currentMantraId, displayLanguage, blockedPackages), loaded at app start, refreshed on every write. Defaults: false / "gayatri" / DEVANAGARI_SANSKRIT / {instagram, facebook, youtube}.
  • NiyamApplication warm-up: MantraRepository.ensureLoaded(this); UserPrefs.ensureLoaded(this).
  • CurrentSadhana keeps its API but reads the snapshot. BlockList.matches() reads the snapshot; HARDCODED_PACKAGES → DEFAULT_PACKAGES.
  • Onboarding writes via OnboardingViewModel (the project's first ViewModel — flow state across 5 screens justifies it).

10. Acceptance criteria

  1. Fresh-install walkthrough on emulator: Welcome → 4 steps → 5 permission screens → Home, all themed, all CTAs orange, permission deep-links unregressed.
  2. Choices take effect: intention filters the 3 cards; chosen mantra + தமிழ் (Tamil) + Instagram-only → Instagram overlay in Tamil, YouTube launches clean.
  3. Persistence across force-stop: straight to Home; overlay choices intact.
  4. Returning-user routing skips onboarding.
  5. Unit tests: curation (5 intentions + unknown-id drop), UserPrefs mapping + defaults, BlockList default/overridden, language labels. Suite grows from 43; all green.
  6. Engine regression: detect→overlay→15s timer→unlock→debounce unchanged.
  7. Visual sign-off by Pranav — built screens match the approved mockup's feel.
  8. APK growth ≤ 1MB (fonts ≈ 600KB).

11. Known risks

  1. Indic serif inconsistency (accepted v1): Devanagari serif guaranteed; Tamil/Telugu/Kannada may render sans in the overlay. Subsetted-font bundling deferred.
  2. First-launch race: blocked-app launch before warm-up completes → defaults once.
  3. Orange contrast: enforced by the fills-only rule.
  4. Playfair has no Devanagari — Latin wordmark only; Devanagari wordmark deferred.
  5. First ViewModel — adds lifecycle-viewmodel-compose; conventional.
