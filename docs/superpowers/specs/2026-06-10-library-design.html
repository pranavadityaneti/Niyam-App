1. Purpose

Browse all 26 mantras, filter by source/length/intention/deity, read any entry in full, and switch the current sadhana with an honest warning. The Browse Library button lands on Home.

In plain English: users can now wander the whole shelf, read anything in their language, and change their practice — with a clear heads-up that a journey starts over if abandoned mid-way.

3. Scope

In: SourceCategory (model + content + validation); LibraryFilters pure function (TDD); LibraryScreen (chip rows + list); MantraDetailScreen + switch dialog (live day count); routes; Home "Browse library" secondary button.

Out: search · favorites · multi-select filters · audio · notifications (SP-7) · paywall gating (SP-6 decides).

4. Architecture

  • SourceCategory enum, lowercase SerialNames; Mantra.sourceCategory with STOTRA default (parser safety net — the asset must still declare it on every entry, test-enforced).
  • LibraryFilters: Selection(category?, length?, intention?, deity?) + LengthBucket.of(seconds) + apply() — AND across non-null dimensions, catalog order preserved.
  • LibraryScreen: four single-select chip rows + LazyColumn rows (serif name, gist, read-time chip, category/deity caption; orange dot = current, green check = completed).
  • MantraDetailScreen(id): byId (unknown → pop); full render; day count via homeStats for the dialog; switch → setCurrentMantra → pop to Home.
  • Routes library, mantra_detail/{mantraId}; Home button navigates.

6. Error handling

Unknown id → pop back. Zero filter results → friendly empty state. homeStats failure → dialog without day count. Switch persistence failure → logged, UI proceeds (snapshot already updated).
